# Конвенции разработки — Центр Цифровизации

## Правило MCP-паритета (обязательное)

**Каждый новый функционал сопровождается инструментами MCP в том же изменении.**
Система строится как контур, в котором ИИ-ассистент видит весь контекст и может
действовать; фича без MCP-инструментов для ассистента не существует.

При добавлении фичи/домена (страница, данные, действия):

1. Добавить инструменты в реестр `TOOLS` в `mcp_server.py`:
   - `read` — сводки и списки нового домена;
   - `write` — изменение его настроек/справочников;
   - `send` — если появляются рассылки (по умолчанию выключены).
2. Больше ничего: гейтинг на странице «Настройки → MCP-инструменты», аудит вызовов
   в журнал операций и скрытие выключенных из `tools/list` работают из общей обвязки.
3. В смоук-тест — вызов новых инструментов через stdio-подпроцесс.

## Границы данных

- **ПДн пациентов**: прямые идентификаторы (ФИО, СНИЛС) никогда не покидают
  приложение — ни через MCP (`PATIENT_MASK`), ни в письмах. Полные данные — только
  в интерфейсе менеджера. Врачи и сотрудники — служебный контекст, не маскируются.
- **Репозиторий публичный**: никаких реальных адресов почты, паролей, имён серверов
  в коде и тестах — всё настраиваемое живёт в БД или env.

## Технические конвенции

- Зависимости не добавляем без необходимости: stdlib-парсеры (SpreadsheetML/xlsx),
  рукописный MCP (stdio JSON-RPC), без ORM.
- Витрина `emd_docs` — сквозная, недельные первички складываются UPSERT'ом;
  «Детализация» обновляет прошлое сама. Не чистить при сбросе выгрузок.
- Значимые операции — `audit(...)` в журнал операций; рассылки — `log_send(...)`
  с kind/subject/period/by_user.
- UI на русском; стили глобальные в `static/style.css` — менять глобально, не точечно.
